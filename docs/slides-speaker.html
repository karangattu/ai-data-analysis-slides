What is Gemini?

Gemini is Google’s family of multimodal AI models that can reason over text, code, and structured data (and more).

Key strengths for SFBBO staff:

Summarization — distill years of survey data

Pattern detection — find migration trends

Code generation — automate data cleaning

Narrative storytelling — write reports and grant summaries

AI vs LLM: What’s the Difference?

AI (Artificial Intelligence) is a broad field:

Any technology that performs tasks humans usually do

Includes chatbots, recommendation engines, image recognition, game-playing bots

LLM (Large Language Model) is a specific type of AI:

Uses text data to understand and generate language

Trained on billions of words from the internet, books, code, etc.

Gemini, ChatGPT, and Claude are all LLMs

Think of it like birdwatching: All LLMs are AI, but not all AI is LLMs—just like all shorebirds are birds, but not all birds are shorebirds!

How LLMs Are Created — Part 1: Training Data

Step 1: Collect massive amounts of text

Billions of pages from the internet, books, research papers, code repositories

Remove spam, personal data, and low-quality text

Curate a balanced dataset covering many topics

Why this matters: - Quality data = better understanding of language patterns - Diverse data = model works across many domains (including ornithology!) - Quantity = model learns more nuanced patterns

How LLMs Are Created — Part 2: Training Process

Step 2: The model learns through prediction

Imagine teaching a bird watcher to identify species. You show examples:

"This small brown bird with a ___ is a House Sparrow."

The model learns: “The blank word should be ‘curved bill’ or ‘striped head’.”

Same concept for LLMs:

  • Show the model: “The Dunlin (Calidris alpina) is a small ___”
  • Model predicts: “shorebird”
  • If wrong, adjust so it gets better next time
  • Repeat billions of times

This teaches the model patterns about what words go together and what makes sense.

How LLMs Are Created — Part 3: Fine-tuning & Safety

Step 3: Make it helpful and safe

Human experts label examples of good vs. bad responses

Train the model to prefer helpful, accurate, honest answers

Add safeguards against harmful outputs

Real example for SFBBO:

  • Train on bird survey best practices
  • Reward: “Here’s how to analyze tern breeding success using SFBBO protocols”
  • Penalize: Made-up conservation statistics

How LLMs Are Created — Summary

The journey from data to AI assistant:

  1. Gather text data
  2. Learn patterns through prediction
  3. Optimize and refine
  4. Add human feedback
  5. Safety testing

The model learns to: - Understand context and meaning - Follow instructions - Admit uncertainty - Write in different styles - Reason through problems

What LLMs Can Do Well

Text understanding & analysis - Summarize SFBBO survey notes - Extract key information from datasets

Writing & communication - Draft reports, grant proposals, newsletters - Generate multiple versions or reframings

Coding & automation - Write Python/R code for data cleaning - Generate SQL queries for databases

Brainstorming & patterns - Suggest research directions - Identify correlations in data

What LLMs Cannot Do

Access real-time information - No internet browsing (unless specifically enabled) - Doesn’t know events after training date - Can’t check current SFBBO monitoring sites

Perform complex math reliably - May make arithmetic errors - Struggles with multi-step calculations - Always verify statistics yourself!

Have personal experiences or emotions - Can’t actually go birdwatching - Can’t feel passion for conservation - Simulates understanding, doesn’t truly “know”

Access external data without you providing it - Doesn’t have access to SFBBO’s internal databases - Can’t pull up historical records on its own - You must share the data to analyze

What LLMs Cannot Do (cont.)

Guarantee accuracy or avoid hallucinations - Can confidently state false information - May invent citations or statistics - Always fact-check outputs!

Understand context you don’t explicitly provide - Doesn’t know SFBBO protocols unless you explain them - Doesn’t know your local bird species distributions - Works best when you give complete context

Make final scientific decisions - LLM suggestions need expert review - Should inform, not replace, your expertise - Always validate with field data

How AI Works (High‑Level)

LLMs are trained to predict the next token in a sequence.

They learn statistical patterns from large corpora to generate useful outputs.

Note

You provide: instructions + data + constraints

Model returns: a best‑guess completion

Tokens, Context, and Cost

Token ≈ a chunk of text (word piece)

Context window = max tokens a model can see at once

Costs scale with input + output tokens

Rule of thumb: \[\text{Cost} \propto \text{input tokens} + \text{output tokens}\]

Prompting Patterns — The Big Three

Master these patterns and you’ll level up your prompts:

Decompose — Break complex tasks into steps

Copy to Gemini
First, summarize species counts by SFBBO monitoring site.
Then, analyze weekly trends for the top 3 shorebird species.
Finally, identify correlations with tide and weather variables.

Prompting Patterns — The Big Three (cont.)

Constrain — Set specific output requirements

Copy to Gemini
Provide exactly 5 bullet points, max 15 words each.
Focus only on shorebird species in the Scolopacidae family.
Use scientific names in parentheses after common names.

Prompting Patterns (cont.)

Verify — Request self-assessment

Copy to Gemini
List your assumptions about the SFBBO dataset.
Rate your confidence (high/medium/low) for each insight.
Note which findings need field verification by SFBBO staff.

Hour 3 — Tooling + Workflows

Typical SFBBO analysis workflow:

  1. Load survey data in a notebook

  2. Use Gemini to generate hypotheses about patterns

  3. Validate with Python/R code

  4. Iterate based on results

Real-World Scenario: Would You Catch This?

You ask Gemini: “My bird survey shows a 400% increase in Snowy Egrets from Jan to Feb. Is this migration or climate change?”

Gemini responds: “This dramatic increase suggests climate-driven habitat expansion.”

What’s missing? (Think 30 seconds…)

Warning

Gemini didn’t ask: Did you check your data entry? Is that 4x real? Could it be observer differences? You need to validate first!

Safe and Reliable Use

Important

Always validate AI outputs against your field expertise!

Cross‑check species IDs and counts with raw data

Keep prompts specific to SFBBO survey protocols

Ask for sources or ecological rationale

Use small, testable analytical steps

Handling Hallucinations

AI can make mistakes with bird data. Mitigation strategies:

Provide data explicitly — don’t assume it knows SFBBO protocols

Ask for uncertainty — “How confident are you?”

Request calculations — have it show the math

Validate with code — quick sanity checks

You are a wildlife data analyst at SFBBO (San Francisco Bay Bird Observatory).

PROJECT: Waterfowl Population Trends Analysis

CONTEXT:
- Data source: SFBBO annual January waterfowl census
- Time period: 2021-2025
- Goal: Inform wetland management and conservation decisions

ANALYSIS REQUIRED:

1. **Trend Analysis**
   - Calculate year-over-year changes for each species
   - Identify any concerning declines (>20%) or notable increases
   - Note the 5-year trend direction for each species

2. **Habitat Comparison**
   - Compare waterfowl use between freshwater and salt pond habitats
   - Identify which species show habitat preferences

3. **Disturbance Assessment**
   - Correlate counts with human disturbance levels
   - Assess impact of water level changes
   - Recommend management interventions

4. **Conservation Recommendations**
